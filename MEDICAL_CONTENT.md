# 醫療與運動衛教內容政策

## 目的與界線

「馬拉松完賽訓練」以遊戲方式介紹馬拉松訓練、恢復、運動傷害警訊與跑步營養。內容只供娛樂與一般衛教，不用於醫療診斷、真實風險預測、個別化課表、營養處方、治療決策或取代合格專業人員評估。

遊戲的體力、受傷風險、速度、關卡時間、道具效果、傷害與健康完賽加分都是平衡機制，不是有效的量表、生理數據或真實建議值。程式測試通過只代表軟體行為符合規則，不等同醫療內容已獲臨床驗證。

## 內容位置

主要衛教文字與來源目前集中在：

- `src/game/data/educationMessages.ts`：事件即時提示與結算重點；
- `src/game/data/educationReminders.ts`：訓練、傷害、營養卡片與紅旗警訊；
- `README.md`：玩法、機制界線與延伸資料來源。

新增內容時請延續資料與 UI 分離的結構，不要把重要醫療文字只寫在場景或樣式檔中。

## 撰寫原則

1. 使用繁體中文、一般人能理解的短句，先說可採取的安全行動，再補充限制。
2. 清楚標示「可能」、「相關」與「因果」的差異；不要把觀察性研究或單一研究寫成確定因果。
3. 不保證任何訓練、補給、護具或治療能避免傷害、提升成績或適用每個人。
4. 不提供個人化診斷、藥物建議、復健處方、精確補給劑量、目標心率或強制訓練進度。
5. 避免責備受傷或未完賽者；把休息、降低負荷與尋求協助呈現為合理選擇。
6. 不使用恐嚇、污名、誇大療效、品牌置入或無法核對的權威背書。
7. 涉及特殊族群、既有疾病、懷孕、藥物、極端氣候或飲食限制時，明確說明一般資訊可能不適用，建議尋求合格專業協助。
8. 只摘要來源所支持的內容，不大量複製受著作權保護的原文、圖表或圖片。

## 來源標準

優先採用：

1. 政府、公衛機關、國際運動組織或專業學會的現行指引；
2. 正式共識聲明、系統性回顧與統合分析；
3. 經同儕審查、方法與適用族群清楚的研究；
4. 具署名、日期與可追溯依據的專業衛教資料。

新聞、社群貼文、品牌行銷、無作者文章與生成式 AI 輸出不能單獨作為醫療主張的依據。AI 可以協助改寫或找出待查問題，但最終內容與引文必須由人員回到原始來源核對。

每項新增或實質修改的主張，Pull Request 應說明：

- 要呈現的主張與目標玩家；
- 原始來源連結、發布年份與查閱日期；
- 來源實際支持的範圍、族群與重要限制；
- 為何目前用詞沒有超出證據；
- 是否涉及立即停下、緊急處置、劑量或其他高風險內容。

連結可用不等於內容仍為最新。重大版本或相關指引更新時應重新核對；本專案目前沒有自動監測來源更新，也不承諾固定年度審查排程。

## 審查與合併

- 一般文字校正仍需確認沒有改變原意。
- 訓練負荷、傷害風險、營養、症狀與恢復等實質主張，至少需要一位能理解相關證據與專案界線的維護者審查後才能合併。
- 紅旗警訊、緊急處置、治療、藥物、補充品或數字建議屬高風險內容；應要求具相關專業能力的人員額外審查，並保留來源與審查日期。
- 若證據不足、來源互相矛盾或無法安全濃縮，應刪除主張、降低確定語氣，或只提示尋求合格專業評估。

維護者審查不代表外部獨立同儕審查、醫療器材認證或任何學會背書。除非有可公開核對的紀錄，不得宣稱內容「經臨床驗證」或「保證安全」。

## 安全與紅旗警訊

一般疲勞不應掩蓋需要立即處理的症狀。涉及下列情況時，內容應明確指示停止運動並依嚴重程度尋求醫療協助：

- 胸痛或胸悶；
- 運動中昏倒、意識混亂或抽搐；
- 嚴重或持續的呼吸困難；
- 高溫環境下的意識改變、昏倒或快速惡化；
- 明顯外傷、無法承重、劇烈疼痛，或其他令人擔心的急性狀況。

緊急服務名稱與電話因地區不同，預設文字應使用「當地緊急醫療服務」，除非內容明確只提供給特定地區。

## 錯誤與更新

若發現衛教內容可能造成傷害、引用錯誤或已過時，請在一般 Issue 提供不含個人健康資料的來源與修正建議。若問題同時涉及可被利用的安全漏洞或玩家資料，請改依 [SECURITY.md](./SECURITY.md) 私下回報。

維護者可先移除或降級高風險內容，再完成查證。修正時應保留清楚的 Git 歷史，不以靜默改寫掩蓋重要內容變動。
