# 參與貢獻

謝謝你願意協助改善「馬拉松完賽訓練」。本專案同時包含遊戲、一般運動衛教、Cloudflare Functions 與 D1 排行榜；提交變更時，請兼顧可玩性、可及性、資料安全與衛教表述。

## 開始之前

- 請先閱讀 [行為準則](./CODE_OF_CONDUCT.md)、[安全政策](./SECURITY.md)、[隱私說明](./PRIVACY.md)與[醫療衛教內容政策](./MEDICAL_CONTENT.md)。
- 安全漏洞或可能暴露玩家資料的問題不得在公開 Issue 或 Pull Request 揭露細節，請依安全政策私下回報。
- 若只是提出想法，可以先開 Issue 說明玩家情境、預期結果與可能影響；專案維護者不保證每一項提案都會採用。
- 請勿提交真實病歷、可識別個人的健康資訊、未經允許的帳密、金鑰或正式環境資料。

## 本機開發

需求為 Node.js 22 以上與 pnpm 10 以上。

```bash
pnpm install --frozen-lockfile
pnpm dev
```

需要測試 Pages Functions 與 D1 時，請使用本機 D1，不要連接或修改正式資料庫：

```bash
pnpm db:migrate:local
pnpm build
pnpm dev:cloudflare
```

本機 `.dev.vars` 可設定開發用 `RATE_LIMIT_SECRET`，但不得提交該檔或任何真實 secret。

## 建議工作流程

1. Fork 專案並從最新的 `main` 建立用途單一、名稱清楚的分支。
2. 先建立或更新能證明問題與預期行為的測試。
3. 保持變更範圍小，避免把無關格式調整、重構與新功能混在同一個 Pull Request。
4. 在本機完成與風險相稱的驗證，再提交 Pull Request。
5. 在說明中列出玩家可見變化、測試結果、手機／桌機檢查方式、資料或 migration 影響，以及仍存在的限制。

提交訊息沒有強制格式，但應使用能描述結果的簡短文字，例如 `fix: 改善手機跳躍輸入`。請勿把密碼、token、玩家資料或正式 D1 匯出內容加入 commit。

## 品質檢查

一般程式變更至少執行：

```bash
pnpm typecheck
pnpm lint
pnpm format
pnpm test
pnpm build
```

影響玩家流程、輸入、響應式版面、Canvas 或分享功能時，另執行：

```bash
pnpm test:e2e
```

影響 Pages Functions 時，另執行：

```bash
pnpm build:functions
```

若有測試未執行或未通過，請在 Pull Request 說明原因，不要以模糊的「應該沒問題」取代驗證結果。

## 各類變更的注意事項

### 遊戲與介面

- 同時檢查手機直式與桌機畫面，確保觸控目標足夠、文字不互相擠壓，操作時不會帶動頁面捲動或縮放。
- 不得只靠顏色傳達體力、風險、狀態或錯誤；保留文字、數值或圖示提示。
- 避免快速閃爍、無法通過的障礙排列，以及會讓低效能裝置顯著掉幀的效果。
- 平衡數值集中放在設定或純函式，不要把相同規則散落在場景與 UI。

### 衛教內容

- 遵循 [醫療衛教內容政策](./MEDICAL_CONTENT.md)，附上可核對的來源、原文所支持的主張與查閱日期。
- 使用一般、非個人化語句，明確區分遊戲數值與真實世界的訓練、營養或醫療建議。
- 不要把相關性寫成因果、把單篇研究寫成定論，或保證某方法可以預防傷害。
- 若修改紅旗警訊、治療、營養劑量或其他高風險內容，請在 Pull Request 中標示，等待具相關專業能力的維護者審查。

### 排行榜、隱私與安全

- API 輸入與輸出都要採明確 allowlist、大小限制與型別驗證；玩家文字在 UI 中不得以不安全 HTML 插入。
- 不要在 client、log、測試 fixture 或錯誤訊息中放入真實 secret、完整 token、IP 或玩家資料。
- 變更收集欄位、公開榜單內容、保存期限或第三方服務時，必須同步更新 [PRIVACY.md](./PRIVACY.md)。
- 防作弊規則屬風險控制，不應宣稱能完全杜絕作弊。

### D1 migration

- 先在全新的本機 D1 與既有 schema 升級路徑各驗證一次。
- schema 變更與清榜／刪除資料應拆成不同、可明確審核的操作；不得在一般 migration 中悄悄加入破壞性資料重設。
- Pull Request 不得執行 `pnpm db:migrate:remote`，也不得假設 Git 部署會自動套用 migration。
- 說明向前／向後相容性、部署順序、回復方式及資料影響。

### 素材與相依套件

- 只提交自行製作、明確允許再散布，或已取得授權的素材，並記錄來源與授權；不得使用品牌 Logo、知名角色、真實運動員肖像或來源不明素材。
- 新增或升級執行期套件時，說明必要性、bundle 影響與授權，並更新 `pnpm-lock.yaml` 及 [THIRD_PARTY_NOTICES.md](./THIRD_PARTY_NOTICES.md)。
- 不要直接複製資料來源的長篇文字或圖片；以自己的文字摘要並連結原始來源。
