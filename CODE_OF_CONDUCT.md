# 社群行為準則

本專案希望提供一個尊重、安全、能共同學習的協作環境。參與 Issue、Pull Request、程式碼審查、Discussion，以及以本專案名義進行的線上或實體互動時，都應遵守本準則。

## 我們鼓勵的行為

- 對不同經驗、能力、身分、文化與觀點保持尊重與好奇。
- 聚焦具體行為、程式碼與證據，提出可採取的建議，不攻擊個人。
- 接受善意回饋、承認錯誤並協助修正。
- 在討論健康、傷害與完賽經驗時使用不責備、不污名的語言。
- 保護玩家、回報者與貢獻者的隱私；範例與測試資料應去識別化。
- 尊重維護者的時間、專案範圍、授權與安全流程。

## 不可接受的行為

- 騷擾、威脅、歧視、羞辱、跟蹤或帶有性意味的不受歡迎言行。
- 人身攻擊、惡意挑釁、持續打斷討論或公開他人私人資訊。
- 冒充他人、操縱排行榜、散布惡意程式，或鼓勵破壞正式服務。
- 在公開管道張貼未修補漏洞、secret、token、IP、病歷或可識別個人的資料。
- 提供危險、誤導或宣稱可取代專業評估的醫療建議。
- 提交未授權素材、抄襲內容、垃圾訊息或與專案無關的宣傳。
- 報復善意提出問題、回報漏洞或指出衛教風險的人。

## 執行與回報

維護者可依情節編輯或移除內容、要求停止行為、暫時限制互動，或永久禁止參與。處置會考量影響、重複性、修正意願與社群安全；本個人／志工專案不保證固定處理時限或正式申訴程序。

一般行為問題可向專案維護者回報。若回報包含個人資料、安全漏洞、威脅或其他不適合公開的細節，請勿建立公開 Issue；請透過[製作者網站](https://sportsmedicine.tw/)或專案擁有者在 GitHub 公開的聯絡方式，先要求私人聯絡管道。安全漏洞另依 [SECURITY.md](./SECURITY.md) 回報。

回報時請提供事件時間、發生位置、相關內容連結或截圖，以及你希望維護者注意的影響。不要為蒐證而擴大衝突或轉傳敏感資料。維護者應盡合理努力限制回報資訊的分享範圍，但無法承諾法律上的絕對保密。

惡意或明知不實的回報本身可能違反本準則；善意但最後未能證實的回報不因此受罰。

## 適用範圍

本準則適用於專案空間，以及個人明確代表本專案或社群時的行為。它不取代 GitHub 的服務條款、平台政策或適用法律。
